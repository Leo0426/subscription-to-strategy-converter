# Leo 策略模板

`leo.yaml` 是从原社区完整 YAML 及其 OpenClash 覆写版本归一化得到的单一策略模板，由 Subflow 分别编译为 Mihomo 和 Surge 配置。

## 使用

1. 在 Subflow 页面填写自己的订阅地址，不要直接在模板中保存订阅 URL。
2. Subflow 会把当前订阅节点写入自动选择、故障转移和手动选择组。
3. 地区组按节点名称匹配；没有匹配节点的地区组及其父级引用会被自动删除。
4. 根据运行环境决定是否启用 `tun.enable`；可用 `python -m app.core.rule_source_audit` 复查远程 RuleProvider。

## 合并原则

- 扫描完整配置：59 份。
- 原始规则：2103 条；非 RULE-SET 内联规则归一去重后：146 条。
- 初始远程规则源 716 个；经过内容审计后保留 508 个，删除 178 个同语义完整副本、30 个连续 404/不可解析的无效源。
- 同名但定义冲突的社区策略组没有机械拼接，而是映射到统一的地区、服务和兜底策略组。
- 广告类规则映射到 `REJECT`，国内和网络基础规则映射到 `DIRECT`，其余规则映射到对应服务组。
- 规则顺序按“拦截 → 国内直连 → 专用服务 → 默认代理 → 兜底”组织，避免广义 Global 规则抢先覆盖 Google、Microsoft、流媒体等细粒度出口。

## 当前质量基线

- 2026-07-20 审计：508 个来源，476 个本轮可用，0 个格式无效，0 个 HTTP 404。
- 初步结构评分：95.95/100（A）；该评分不替代语义准确率、覆盖率和长期新鲜度验证。
- 仍保留 403 或瞬时失败来源，避免把抓取环境限制误判为永久失效。
- Surge 会跳过无法转换为文本规则的 MRS 来源；Mihomo 可完整使用这些来源。平台差异应结合生成结果中的 warning 评估。
