# 领域文档

本仓库采用单一上下文布局。

## 探索规则

工程类 skills 开始工作前：

1. 读取根目录 `CONTEXT.md`
2. 读取与当前操作区域相关的 `docs/adr/`
3. 使用 `CONTEXT.md` 定义的领域术语
4. 如果方案与 ADR 冲突，必须明确标注，不得静默覆盖

若领域文件不存在则静默继续，不主动创建；在确实需要补充领域语言或决策时，交由 `domain-modeling` 或 `decision-forge` 处理。

## 领域语言

Issue 标题、测试名称、重构提案和架构决策应优先使用 `CONTEXT.md` 中定义的术语。发现缺失概念时，应记录为领域模型空缺，不自行引入含义重叠的同义词。
