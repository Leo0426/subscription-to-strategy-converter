SUBFLOW GUIDE

发布可靠的
客户端策略订阅

Subflow 不要求你先理解 YAML、provider 或策略图。完成四个步骤后，即可获得 Clash / Mihomo 与 Surge 长期订阅地址。

01导入订阅
02目标与模板
03Claude 路由
04验证并发布

01 · 导入订阅

打开 Subflow，粘贴授权的机场订阅 URL，然后点击“连接订阅”。系统只解析节点，不会在此阶段发布或修改客户端配置。

源订阅 URL 保存在本机 SQLite Profile 中，Profile 列表不会回显它。

02 · 确认模板推荐

系统先理解你的目标客户端，再给出模板组合。推荐结果可审计，也可以通过“更换模板”手动替换。

目标候选范围兼容要求
Clash / Mihomo全部含 Claude 策略的模板存在可识别的 Claude 规则子图
Surge严格兼容模板完整 provider、规则类型和节点协议可原样编译

03 · 指定 Claude 出口

从真实订阅节点中选择一个稳定出口。Subflow 只修改模板的 Claude 策略子图：

已有 Claude 专属组

把选择的出口放在组成员首位，其他成员原样保留。

共用 AI / OpenAI 组

创建独立 Claude 组，将原目标保留为回退。

provider URL、规则顺序、DNS/TUN 与非 Claude 策略不会改变。

04 · 验证并发布

进入最后一步后，Subflow 自动构建两个目标并执行分析。只有 Clash 与 Surge 都能安全生成时，“发布 Profile”才会启用。

/subscribe/<id>?token=…&target=clash
/subscribe/<id>?token=…&target=surge

将两个地址分别添加到客户端订阅管理即可。

Profile 管理

再次打开 Subflow 时，有 Profile 就先进入管理首页；没有 Profile 则自动开始创建。

复制链接创建该 Profile 的浏览器保存 token 后可直接复制。
编辑使用 token 授权读取原草稿，复用相同四步流程。
更新清空旧产物缓存，避免配置变化后回退到旧语义。
上游故障只回退当前目标自己的最后成功产物。

配置检查器

右上角“检查器”提供四个按需工具，不打断主流程：

概览 / 问题

查看节点、策略统计与阻止发布的分析结果。

测试 / 源码

模拟域名命中路径，或查看最终 Clash、Surge 配置。
