Subflow · 代理策略工作区

把订阅链接变成
可理解的策略配置

Subflow 将机场订阅节点与模板、规则集合成一个可检查、可模拟、可编译的策略工作区，导出为 Mihomo / Clash / sing-box / Surge 配置文件。

订阅 URL 机场提供
ProxyNode IR 归一化节点
PolicyWorkspace 模板 + 规则
编译输出 Mihomo / sing-box…
296 条规则集

8 个分类，AI / 流媒体 / 社交通讯 / 广告拦截 / 国内直连 / 代理规则 / 网络基础 / 其他，去重后保留最优来源。

108 个社区模板

内联浏览器直接搜索预览，一键应用，免切页面。

规则分析 + 流量模拟

自动检测缺失 provider、策略组环。输入域名追踪命中路径，所见即所得。

Profile 持久化

生成 token 保护短链，客户端直拉自动重编译，上游断源时回退最后成功产物。

快速启动

Docker（推荐）

git clone <repo>
docker compose up

打开 http://127.0.0.1:8000。Profile 数据库自动写入 ./data/subflow.db。

本地开发

uv sync
uv run uvicorn app.main:app --reload
依赖 Docker 模式只需 Docker，本地开发需要 Python 3.12+ 与 uv。无其他外部服务依赖。

两种使用模式

GET /

傻瓜模式

粘贴订阅链接，秒得可直接导入 Mihomo / Clash 的订阅 URL。无状态，不落库。

GET /advanced

高级工作台

完整策略编辑器：模板、规则编排、分析、模拟、编译、持久化 Profile，全在一个页面。

傻瓜模式用法

1
粘贴订阅链接

在首页"机场订阅 URL"输入框粘贴你的订阅地址（Clash / Mihomo 格式均可）。

2
自动生成链接

下方"订阅链接"框自动填充，点击"复制"按钮。

3
导入客户端

把复制的链接填入 Mihomo / Clash Verge 的订阅管理，更新即可。

高级工作台详解

打开 /advanced，按以下五步完成一次完整的策略编译。

工作台 · 步骤一

加载节点

1
填入订阅 URL

顶部"订阅 URL"输入框填入机场订阅地址。

2
测试订阅（可选）

点击"测试订阅"，检查节点数量与可用性，不会写入任何状态。

3
生成配置

点击"生成配置"，节点解析为 ProxyNode IR，与当前选中的模板合成 PolicyWorkspace，配置实时出现在"配置预览"面板。

工作台 · 步骤二

选择模板

"内置预设"下拉列表提供三类模板：

模板说明
minimal仅节点，无规则集，适合测试节点连通性
powerfullz完整分流，含 AI / 流媒体 / 广告拦截 / 国内直连等策略组
community:…社区贡献配置，从下方社区浏览器选取

社区浏览器直接内联在"内置预设"模块下方。支持关键词搜索，点击任一条目可在右侧预览策略组结构，确认后点击"应用"。

选 powerfullz 后，面板下方会出现"Powerfullz 选项"，可开关 fakeip、TUN、IPv6、负载均衡等能力。
工作台 · 步骤三

编排规则（可选）

切换到"规则编排"面板，在模板规则的基础上追加定制规则。

1
从规则集目录选取

右侧规则集目录按 8 个分类展示 296 条规则集，点击即追加到当前策略。每条规则可单独指定目标策略组。

2
输入自定义规则

"自定义规则"文本框直接输入原始规则行，例如 DOMAIN-SUFFIX,example.com,DIRECT。

3
重新生成

规则变更后，点击"生成配置"重新编译。"清空定制"按钮可一键撤销所有自定义规则。

工作台 · 步骤四

验证

规则分析

切换到"规则分析"面板，Subflow 对当前 PolicyWorkspace 做结构检查，报告：

  • 缺失的 rule-provider（引用了但未定义）
  • 缺失的目标策略组
  • 重复规则
  • 策略组依赖环
  • 不可达策略组

流量模拟

切换到"流量模拟"面板，输入域名（如 openai.com）或 IP，Subflow 沿规则表逐条匹配，输出命中的规则与最终策略组。

注意： RULE-SET 和 GEOIP 类规则当前做结构模拟，不做真实内容匹配。
工作台 · 步骤五

选择目标，导出

右上角"目标客户端"下拉切换格式，配置在"配置预览"面板实时重新编译。

格式状态说明
Mihomo YAML 稳定 主路径，完整语义支持
Clash YAML 稳定 兼容旧版 Clash 客户端
sing-box JSON 实验性 不保证与 Mihomo 完整语义对等
Surge conf 实验性 不保证与 Mihomo 完整语义对等

点击"复制"或"下载"取得配置文件。"完整输出" / "策略片段" / "自动预设源"三个视图可切换查看不同层级的输出内容。

Profile：长期托管订阅

不想每次手动更新配置？用 Profile 把工作台的配置变成一个客户端可以直拉的订阅链接。

1
保存为长期订阅

工作台编译好配置后，点击"保存为长期订阅"。Subflow 把当前参数（订阅 URL、模板、规则、目标）持久化到数据库。

2
获取长期链接

系统生成形如 /subscribe/<id>?token=… 的链接，复制后填入代理客户端的订阅管理。

3
客户端自动更新

客户端每次拉取时，Subflow 自动重新获取上游订阅、重编译配置并返回最新结果。若上游暂时不可用，自动回退到最后一次成功产物。

已保存的所有 Profile 可在工作台右侧"已保存订阅"面板中管理。

目标格式说明

Mihomo 和 Clash 是同一编译路径，Mihomo 是推荐选项。sing-box 与 Surge 经过独立适配器输出，当前为实验性支持，部分高级特性（RULE-SET 内容匹配、策略组复杂继承）可能与 Mihomo 行为不完全一致。

环境变量

变量默认值说明
SUBFLOW_DB_PATH ./data/subflow.db Profile 数据库路径，目录不存在时自动创建
